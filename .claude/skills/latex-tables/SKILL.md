# LaTeX Table Export Skill

Export statistical tables as publication-ready LaTeX fragments.

**Input:** `<args>` -- a DataFrame variable, table description, or path to data.

---

## When to Trigger

Use this skill whenever the user wants to:
- Export summary statistics or regression results to `.tex`
- Format tabular output with booktabs, siunitx, threeparttable, or scalebox
- Build or refactor a table export function in Python
- Convert a pandas DataFrame or regression result into a `.tex` file
- Reference `\input{}` of a table fragment
- Format column alignment, significance stars, fixed effects rows, or R-squared reporting
- Generate or post-process Stata `esttab` output (event study, IV/2SLS, regression tables)

**Do NOT trigger** for general LaTeX document formatting, figure export, or regression modeling unrelated to table output.

---

## Core Conventions (Invariant)

These apply to every table, every paper, regardless of whether it was generated by Python or Stata:

### 1. No Float Wrapper

The `.tex` fragment never contains `\begin{table}`, `\caption`, or `\label`. Those live in the main document that `\input{}`s the fragment.

```latex
% CORRECT — fragment file contains tabular (+ optional scoping braces):
\begin{tabular}{l S[table-format=1.3] S[table-format=5.0]}
\toprule
Variable & {Mean} & {N} \\
\midrule
Price & 0.432 & 1423 \\
\bottomrule
\end{tabular}
```

```latex
% ALSO CORRECT — Stata fragment with scoped \def\sym:
{
\def\sym#1{\ifmmode^{#1}\else\(^{#1}\)\fi}
\begin{tabular}{l*{4}{c}}
...
\end{tabular}
}
```

```latex
% WRONG — never include this in the fragment:
\begin{table}[ht!]
\caption{Summary Statistics}
\label{tab:sumstats}
...
\end{table}
```

### 2. Booktabs Always

Use `\toprule`, `\midrule`, `\bottomrule`. **Never** `\hline`.

### 3. Caption and Label Live in Main Doc

The main `.tex` document wraps the fragment:

```latex
\begin{table}[ht!]
\caption{Summary Statistics} \label{tab:sumstats}
\input{Tables/02Aug2025/sumstats.tex}
\begin{singlespace}\footnotesize
\noindent \textsc{Notes}: Description of the table...
\end{singlespace}
\end{table}
```

### 4. Consistent Column Alignment Within a Paper

Two valid styles exist — pick one and use it throughout the paper:

**Python-generated tables (siunitx):**

| Format | Use for |
|--------|---------|
| `S[table-format=5.0]` | Integers (counts) |
| `S[table-format=6.2]` | Dollar amounts |
| `S[table-format=1.3]` | Rates, shares, fractions |
| `S[table-format=-1.3]` | Signed coefficients |
| `S[table-format=-1.3, table-space-text-post={***}]` | Coefficients with significance stars |

Wrap text column headers in `{}` to escape siunitx parsing: `{Variable}`, `{(1)}`.

**Stata-generated tables:** Use `l` (left-aligned) or `c` (center-aligned) columns. `esttab` handles spacing internally. See the Stata section below for conventions.

### 5. Notes Placement

- **Substantive notes** (sample description, variable definitions) always go in the main doc.
- **Stata star-legend footnotes** (`\footnotesize \sym{*} \(p<0.10\)...`) may stay in the fragment when `\def\sym` requires scoping. This is standard `esttab` output.

## Python Export Pipeline

### Basic DataFrame Export

```python
import pandas as pd
from pathlib import Path
from datetime import datetime

def export_latex_table(
    df: pd.DataFrame,
    output_path: Path,
    col_formats: dict[str, str] | None = None,
    int_cols: list[str] | None = None,
    float_fmt: str = ".3f",
) -> None:
    """Export DataFrame as a booktabs LaTeX fragment with siunitx alignment.

    Parameters
    ----------
    df : DataFrame to export
    output_path : Path to .tex file
    col_formats : dict mapping column name to siunitx format string
                  e.g. {"N": "5.0", "Mean": "1.3"}
    int_cols : columns to format as integers (no decimals, with commas)
    float_fmt : default float format for unspecified columns
    """
    output_path.parent.mkdir(parents=True, exist_ok=True)

    # Build column format string
    if col_formats is None:
        col_formats = {}

    col_spec_parts = ["l"]  # first column is left-aligned text
    # Extract decimal places from float_fmt (e.g., ".3f" -> "3", ".10f" -> "10")
    default_decimals = float_fmt.lstrip(".,").rstrip("f")
    for col in df.columns[1:]:
        if col in col_formats:
            col_spec_parts.append(f"S[table-format={col_formats[col]}]")
        else:
            col_spec_parts.append(f"S[table-format=1.{default_decimals}]")

    col_spec = " ".join(col_spec_parts)

    # Build header row (wrap in {} for siunitx)
    header = " & ".join(f"{{{col}}}" for col in df.columns) + r" \\"

    # Build data rows
    rows = []
    for _, row in df.iterrows():
        parts = []
        for i, (col, val) in enumerate(row.items()):
            if i == 0:
                parts.append(str(val))
            elif int_cols and col in int_cols:
                parts.append(f"{int(val):,}" if pd.notna(val) else "")
            elif pd.isna(val):
                parts.append("")
            else:
                parts.append(f"{val:{float_fmt}}")
        rows.append(" & ".join(parts) + r" \\")

    # Assemble
    lines = [
        f"\\begin{{tabular}}{{{col_spec}}}",
        "\\toprule",
        header,
        "\\midrule",
        *rows,
        "\\bottomrule",
        f"\\end{{tabular}}",
    ]

    output_path.write_text("\n".join(lines) + "\n")
    print(f"Saved: {output_path}")
```

### Regression Table Export

Complete function for multi-specification regression tables with coefficients, standard errors, significance stars, fixed effects indicators, N, and R-squared:

```python
from dataclasses import dataclass


def sig_stars(pval: float) -> str:
    if pval < 0.01:
        return "***"
    elif pval < 0.05:
        return "**"
    elif pval < 0.10:
        return "*"
    return ""


@dataclass
class RegSpec:
    """One column (specification) in a regression table."""
    label: str                          # column header, e.g. "(1)"
    coefs: dict[str, float]             # variable name -> coefficient
    ses: dict[str, float]               # variable name -> standard error
    pvals: dict[str, float]             # variable name -> p-value
    fe: dict[str, str] | None = None    # fixed effect name -> "Yes"/"No"
    n: int = 0
    r2: float = 0.0


def export_regression_table(
    specs: list[RegSpec],
    var_order: list[str],
    var_labels: dict[str, str] | None = None,
    output_path: Path = Path("reg_table.tex"),
    coef_fmt: str = ".3f",
) -> None:
    """Export a multi-column regression table as a booktabs LaTeX fragment.

    Parameters
    ----------
    specs : list of RegSpec, one per column
    var_order : variable names in display order
    var_labels : optional mapping from variable name to display label
    output_path : path to .tex file
    coef_fmt : format string for coefficients and SEs
    """
    output_path.parent.mkdir(parents=True, exist_ok=True)
    if var_labels is None:
        var_labels = {}
    ncols = len(specs)

    # Column spec: label column + one S column per spec
    star_fmt = f"-1.{coef_fmt.lstrip('.').rstrip('f')}"
    s_col = f"S[table-format={star_fmt}, table-space-text-post={{***}}]"
    col_spec = "l " + " ".join([s_col] * ncols)

    lines = [f"\\begin{{tabular}}{{{col_spec}}}", "\\toprule"]

    # Header row
    header = " & ".join([" "] + [f"{{{s.label}}}" for s in specs]) + r" \\"
    lines.append(header)
    lines.append("\\midrule")

    # Coefficient rows (coef line + SE line per variable)
    for var in var_order:
        label = var_labels.get(var, var)
        coef_parts = [label]
        se_parts = [""]
        for s in specs:
            if var in s.coefs:
                stars = sig_stars(s.pvals[var])
                coef_parts.append(f"{s.coefs[var]:{coef_fmt}}{stars}")
                se_parts.append(f"({s.ses[var]:{coef_fmt}})")
            else:
                coef_parts.append("")
                se_parts.append("")
        lines.append(" & ".join(coef_parts) + r" \\")
        lines.append(" & ".join(se_parts) + r" \\[0.5em]")

    lines.append("\\midrule")

    # Fixed effects rows
    fe_names: list[str] = []
    for s in specs:
        if s.fe:
            fe_names.extend(k for k in s.fe if k not in fe_names)
    for fe in fe_names:
        parts = [fe]
        for s in specs:
            parts.append(f"{{{s.fe.get(fe, 'No')}}}" if s.fe else "{No}")
        lines.append(" & ".join(parts) + r" \\")

    # N and R-squared
    n_parts = ["{N}"] + [f"{{{s.n:,}}}" for s in specs]
    r2_parts = ["{R$^2$}"] + [f"{s.r2:.3f}" for s in specs]
    lines.append(" & ".join(n_parts) + r" \\")
    lines.append(" & ".join(r2_parts) + r" \\")

    lines.extend(["\\bottomrule", "\\end{tabular}"])
    output_path.write_text("\n".join(lines) + "\n")
    print(f"Saved: {output_path}")
```

## Stata Table Export

Stata `esttab` output uses its own conventions that differ from Python/siunitx tables. Both formats are acceptable — use whichever matches the project's existing tables for consistency.

### esttab Event Study / Regression Table

Standard `esttab` output for economics papers (event study example):

```stata
esttab m1 m2 m3 m4 using "tablename.tex", replace ///
    booktabs fragment                                ///
    cells(b(star fmt(3)) se(par fmt(3)))             ///
    star(* 0.10 ** 0.05 *** 0.01)                   ///
    stats(ymean N avgeffect jointeq jointplacebo,    ///
          fmt(%9.3f %9,0gc %9.3f %9.3f %9.3f)       ///
          labels("Dependent Variable Mean"           ///
                 "Observations"                      ///
                 "Average Total Effect"              ///
                 "Joint Eq. Effects"                 ///
                 "Joint Sig. Placebo"))              ///
    indicate("County + Year FE = *.county *.year"    ///
             "Demographics = demo_*"                 ///
             "Economics = econ_*"                    ///
             "Housing = housing_*")                  ///
    mlabels("(1)" "(2)" "(3)" "(4)")                 ///
    collabels(none) nomtitles                        ///
    varlabels(, blist(beta_0 "\addlinespace"         ///
                      beta_1 "\addlinespace"))       ///
    nonotes
```

### esttab IV / 2SLS Table

IV tables typically combine first stage and second stage in one table, with different dependent variables as column headers. Two header rows: column numbers on top, dependent variable names below.

```stata
* First stage
eststo fs: reghdfe endogenous_var instrument controls, ///
    absorb(county year) cluster(county)

* Second stage (multiple outcomes)
foreach y in outcome1 outcome2 outcome3 {
    eststo `y': ivreghdfe `y' (endogenous_var = instrument) controls, ///
        absorb(county year) cluster(county)
}

* Export with IV-specific stats
esttab fs outcome1 outcome2 outcome3 using "iv_table.tex", replace ///
    booktabs fragment                                               ///
    cells(b(star fmt(3)) se(par fmt(3)))                            ///
    star(* 0.10 ** 0.05 *** 0.01)                                  ///
    stats(r2 r2_a N fstat mpfstat mpcrit,                           ///
          fmt(%9.3f %9.3f %9,0gc %9.1f %9.1f %9.2f)                ///
          labels("R-squared" "Adjusted R-squared" "Observations"    ///
                 "F-Statistic" "MP F-stat"                          ///
                 "MP 5\% Critical Value"))                          ///
    indicate("County + Year FE = *.county *.year"                   ///
             "Controls = controls_*")                               ///
    mgroups("First Stage" "Second Stage",                           ///
            pattern(1 1 0 0))                                       ///
    mtitles("Endogenous Var" "Outcome 1" "Outcome 2" "Outcome 3")  ///
    collabels(none) nonotes
```

Key IV table conventions:
- **Two header rows**: column numbers `(1)...(k)` on top, dependent variable names below
- **First stage in column 1** with instrument coefficient and F-statistic
- **Second stage in remaining columns** with endogenous variable coefficient and MP F-stat
- **Report both F-stat types**: first-stage F for column 1, Montiel-Pflueger (or Kleibergen-Paap) F for 2SLS columns
- **MP critical value row**: Stock-Yogo or MP 5% critical value for weak instrument test
- **Empty cells** where stats don't apply (e.g., no MP F-stat for first stage) — leave blank, don't write "." or "N/A"

### Post-Processing Stata Output

If a Stata `.tex` file was exported without `fragment`, strip the wrapper:

```python
def strip_table_wrapper(tex_path: Path) -> None:
    """Remove \\begin{table}...\\end{table} wrapper, keeping only the tabular."""
    text = tex_path.read_text()
    # Extract tabular environment (including outer braces if present)
    if text.strip().startswith("{"):
        start = text.index("{")
        end = text.rindex("}") + 1
    else:
        start = text.index(r"\begin{tabular")
        end = text.index(r"\end{tabular}") + len(r"\end{tabular}")
    tex_path.write_text(text[start:end] + "\n")
```

## Workflow

### Step 1: Read CLAUDE.md

Check the project's output path conventions and table naming.

### Step 2: Determine Table Type and Style

- **Summary statistics**: counts, means, SDs. Use appropriate decimal places per variable type.
- **Regression table**: coefficients, SEs, stars, fixed effects indicators, N, R-squared.
- **Event study table**: coefficient rows with `\addlinespace`, placebo/joint test rows.
- **IV / 2SLS table**: first stage + second stage, two header rows, F-stats and MP critical values.
- **Custom table**: follow user specifications.

### Step 3: Build and Export

1. Construct the DataFrame or Stata estimates
2. Export using the fragment-only convention (match project's column alignment style per Convention #4)
3. Print the output path

### Step 4: Verify

Read back the `.tex` file and confirm:
- [ ] No `\begin{table}` wrapper (outer `{}` braces for `\def\sym` scoping are OK)
- [ ] Uses `\toprule`/`\midrule`/`\bottomrule` (no `\hline`)
- [ ] Column alignment consistent with project style (siunitx `S` or Stata `l`)
- [ ] If siunitx: column headers wrapped in `{}`
- [ ] If Stata: `\sym` macro defined and in scope
- [ ] Notes/captions in main doc (Stata star-legend footnotes are acceptable when `\def\sym` requires scoping)
